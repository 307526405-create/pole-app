## 六、技术方案

- **前端：** uni-app(Vue3) → 微信小程序
- **后端：** Python FastAPI + SQLite
- **数据：** jolpica自部署(主) + Fast-F1(备份) + 本地缓存(兜底)
- **资讯：** Motorsport.com RSS自动抓取
- **部署：** 阿里云ECS ¥200-400/月
- **开发：** Claude Code写代码，你管验收

## 七、产品功能

### MVP（8个模块）

| 模块 | 内容 | 数据来源 |
|------|------|---------|
| 🎯 竞猜 | 每站猜杆位+领奖台+最快圈 | 自建 |
| 🏆 排名 | 赛季+迷你赛季排行榜 | 自建 |
| 📅 赛程 | 24站+赛道详情 | jolpica |
| 📊 积分 | 车手+车队积分榜 | jolpica |
| 👤 我的 | 积分+准确率+历史 | 自建 |
| 📤 分享 | 竞猜卡片分享到微信群 | 自建 |
| 📰 快讯 | 赛车新闻标题1-2条/天 | RSS |
| 💎 Pro | 圈速对比+历史数据+无广告 | 自建(P2) |

### 不做
社区、商城、实时数据、新闻稿、WEC/WRC竞猜(无API)

## 十、关键风险

- **最大风险：** 竞猜参与度不足。解决方案：微信群首发+群内竞猜文化培育
- **剧透风险：** 推送毁了比赛回放。解决方案：防剧透模式
- **空窗期：** F1 12-2月无比赛。解决方案：快讯+历史回顾+冬测
- **数据源：** jolpica可能不稳定。解决方案：四层备份
